# MARS-CL — live browser demo

Interactive demonstration of the headline result: class-incremental learning
with **zero stored samples** (semantic word anchors + parametric sleep on a
frozen random backbone). The exact PyTorch-trained weights run in plain
JavaScript; on load the page recomputes a feature vector and compares it
against the PyTorch-exported value (parity check).

## Run locally

```bash
# 1. one-time: train 1 seed (H1b k16 config) and export weights (~minutes on GPU)
python scripts/download_glove.py          # if not done yet
python scripts/export_demo_weights.py     # writes mars_demo_weights.json here

# 2. serve this folder (fetch() does not work from file://)
cd demo/mars_cl_demo
python -m http.server 8000                # -> http://localhost:8000
```

`--smoke` on the export script gives a fast low-quality model for testing
the plumbing.

## Files

- `index.html` — UI (no build step, no dependencies)
- `mars_core.js` — the full forward pass in pure JS (conv/BN/pool/linear,
  cosine routing to word anchors, per-class pods); unit-tested against numpy
- `mars_demo_weights.json` — generated by `scripts/export_demo_weights.py`
  (weights, per-task snapshots, test images, parity vector)

## Hosting note

The weights JSON (~4 MB) can be committed; the demo then works on GitHub
Pages out of the box (Settings → Pages → deploy from branch `main`, path `/`),
at `https://<user>.github.io/m-a-r-s/demo/mars_cl_demo/`.
